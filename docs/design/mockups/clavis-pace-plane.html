Pace Plane — fitting it in & keeping it agent-agnostic

The pace plane is the one form that does not care how many windows an agent has. Both axes are normalized 0..1 (time elapsed, budget spent), so a window of any unit (tokens, requests, messages) and any length (5h, 7d, 1d) collapses to a single point. The diagonal is "perfectly even spend". Any number of dots share the same chart — that is exactly the agent-agnostic property we need.

1 — Glanceable in the status bar, detail on demand

A 22 px mini-plane lives in the status bar, always visible: just the diagonal and the window dots, no labels. Click it to slide in the labelled panel. Nothing in the conversation chrome needs more than a tiny square.

2 — Same component, any agent, any number of windows

UI code never names a provider or assumes two windows — it just plots report.Windows. A bridge that reports one daily cap, or three tiered windows, renders through the identical control.

3 — The agent-agnostic contract (Session group)

A new neutral event in the AgentStreamEvent family. The bridge fills WindowStart from the plan it knows; the UI derives everything (elapsed, expected, pace) purely from the numbers — no provider knowledge leaks past the bridge.

[<Sealed>]
type AgentLimitWindow(name: string, used: float, total: float, unit: string,
                       windowStart: DateTimeOffset, resetsAt: DateTimeOffset) =
    member _.Name = name          // "5-Hour", "Weekly", "Daily"…
    member _.Used = used          // abstract budget units
    member _.Total = total
    member _.Unit = unit          // "tokens" | "requests" | "messages" — display only
    member _.WindowStart = windowStart
    member _.ResetsAt = resetsAt

[<Sealed>]
type AgentUsageReport(sessionId: Guid, windows: IReadOnlyList<AgentLimitWindow>) =
    inherit AgentStreamEvent(sessionId)
    member _.Windows = windows

// Derived UI-side, pure (no provider knowledge):
//   timeFrac = (now - WindowStart) / (ResetsAt - WindowStart)
//   spentFrac = Used / Total
//   pace = spentFrac vs timeFrac   →  dot above diagonal = overspending
