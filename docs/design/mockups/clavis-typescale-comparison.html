Event log machine-dense · the hardest case for a loose scale
Command palette selectable list · name + kind + shortcut
Settings cards prose-ish · descriptions legitimately want to read well
What to look for. In the event log, the document's 16px scale shows only a handful of lines — a log wants density, so the compact tier wins. In settings, the 16px body actually reads better for the descriptions — prose wants room. That split is the whole decision: a single scale can't serve both, which is why the "compact UI scale" option keeps prose (agent/body) generous while tightening chrome (label/mono/meta/micro). Pick a column per panel and notice which feels right — that tells us whether to codify the dual scale, add switchable density tiers, or raise everything to 14/16.
