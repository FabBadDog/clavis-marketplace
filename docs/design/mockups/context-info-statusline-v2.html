CLAVIS v2 — chosen direction, refined to spec

Final design from your decisions. Placeholders are plugin-contributed & namespaced with IntelliSense; components are bar / limitPlane / badge / microstat with a {value:format} transform (e.g. {agent.name:uppercase}). New bar style (label in front, dark full-length track, no %). Top bar: per-panel left title + minimal right agent cluster with Claude mode colors, both configurable & faded on switch. The status-line editor is a new panel; the input-box style becomes a shared control.

01 Placeholder grammar & plugin contribution every plugin owns its namespace

AGrammar
{git.branch} — value
{agent.name:uppercase} — value : format
{time.now:HH:mm} — value : .NET format
{bar:agent.contextPercent} — component : value
{badge:agent.mode} — component : value
{badge:time.now:HH:mm} — component : value : format (composable)
{microstat(arrow-up):turn.runtime} — component(arg) : value
{limitPlane} — component, no value

Leading name decides: a registered component treats the tail as the value; otherwise the name is a value and the tail is a format/transform. Components and formats compose — a component's value can itself carry a format, so {badge:time.now:HH:mm} renders a badge holding the time formatted HH:mm. Unknown tokens render verbatim.

BPlugin-contributed namespacesextensible
agent.*  ◂  ClaudeBridge / agent facade
git.*  ◂  git provider plugin
sys.*  ◂  system-metrics plugin
turn.* / context  ◂  conversation plugin
cwd.* / clavis.* / time.*  ◂  host / core providers

Each plugin registers a PlaceholderProvider (namespace + token descriptors + value resolver) on the bus. The engine aggregates every registration for resolution and IntelliSense — so a new plugin's tokens appear in the catalog automatically.
