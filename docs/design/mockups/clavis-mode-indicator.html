03 Decisions to make naming, color, and the cycle

"Manual" vs. "None"

You called the default mode manual. The catalog id is default and it currently displays as "None" - and today the badge is hidden in that mode.

I'd rename the display to "Manual" and always show it: "None" reads as "no mode set", but it's really the deliberate ask-every-time mode. One label change in ClaudeCatalog.

Always visible

The current chip disappears in default mode. For a mode you can cycle blind with a shortcut, the indicator should always be on screen so a stray Shift+Tab is never silent.

The segmented rail enforces this for free - the active segment always lights up.

Shift+Tab = cycle

Bound by default to shift+tab via the keymap, publishing a "cycle mode" command that advances to the next mode and wraps. Re-bindable like any Clavis shortcut.

An open question: cycle all four, or match Claude Code and cycle a subset? I'd cycle all four in the fixed order below.
