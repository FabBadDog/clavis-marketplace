CLAVIS / badge styles

The three badge shapes in the app today, plus five proposed unified styles. Every option keeps the house rules: square corners only (a dot may be a circle), content before chrome, and colour driven by a theme resource key - so one shared Badge style + IconBadge.create factory can replace all three, configured by an accent key and an optional fill.

Proposed unified styles - pick one

Each is shown across the same accent set (clavis / human / input / warn / error / secondary) so you can compare. Whichever you choose becomes the single Badge style; the accent is a resource key, so kind/level/scope all map onto it.

A  ·  Outlined chip

1px border in the accent colour, transparent fill, accent text. Crisp and low-ink; reads clearly on any panel without a filled block.

B  ·  Left accent bar

A 2-3px vertical bar in the accent colour beside neutral text - maximal content-before-chrome. The colour cues the category while the label stays highly readable.

MESSAGE ALIAS AGENT SKILL ERROR CMD
C  ·  Dot + label

A small filled circle in the accent colour + dim label. Reuses the Clavis dot idiom (status dots, the breathing pixel) so badges feel part of the same family. Circle is allowed; the rule only forbids rounded rectangles.

MESSAGE ALIAS AGENT SKILL ERROR CMD
D  ·  Solid fill, dark text

Square block fully filled in the accent colour with near-black text. The highest-emphasis option - good when a badge should grab attention (errors, active states), perhaps less so for dense lists.

MESSAGE ALIAS AGENT SKILL ERROR CMD
E  ·  Underline accent

Neutral text with a 2px accent underline - directly mirrors the SegmentedSelector selected-underline already in Clavis, so the badge language stays consistent with the rest of the UI. Lightest footprint after the bar.
