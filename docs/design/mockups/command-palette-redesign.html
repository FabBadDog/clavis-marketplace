CLAVIS Command Palette - Redesign Explorations

Five directions for the command-palette popup (rendered today through the shared SelectorWindow). Real default-theme palette, three voices: Rajdhani (chrome), Inter (prose), JetBrains Mono (machine).

Each concept attacks the same three issues. Selected row in every mock is deep-research so you can compare how each handles a long skill description.

Problem 1
No sign that skills are still loading - they arrive async after the popup can open.
Problem 2
Too many lines & boxes - an outer frame, a rule under the input, and a filled badge box on every row.
Problem 3
Long skill descriptions overflow / dominate the row and are hard to scan.

1Quiet Columns

least change Keep the familiar columnar scan but subtract chrome: badge box → a single kind dot (a circle), drop the rule under the input (a blue caret marks focus instead), selection becomes a 2px primary left edge + brighten (the design-language list-row treatment) rather than a wash. Descriptions truncate to one line. A footer status line carries the breathing loading dot.

2Two-line Cards

content-first No columns, no boxes at all. Each result is a small block: name + shortcut on the primary line, source · description dim below. The long description gets a whole line to breathe and wraps to two before clipping, so skills are finally readable. Kind is a leading dot. This is the boldest "content before chrome" reading.

3Grouped by kind, description on focus

scannable Rows carry only name + shortcut with a 2px coloured kind edge; commands are grouped under tiny section heads. The full description shows only for the highlighted row, in a detail strip at the bottom - so long skill descriptions stop competing for space and the list stays dense and fast to scan. Loading shows as a pending group.

4Master / Detail

richest A narrow result list on the left (name + kind dot + shortcut only), a detail pane on the right showing the highlighted command's full, untruncated description + kind + source. 5Spotlight minimal

minimal Stripped to the essentials, macOS-Spotlight-like. Kind is a 2px coloured tick on the selected row only (zero per-row boxes). Shortcut is a faint surface chip; description is dim trailing text, single line. The whole thing leans on the search field. A single footer combines the breathing loading dot, a result count, and key hints.
