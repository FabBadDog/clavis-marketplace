Clavis — Design System

The refined language. Same palette, voices and motion as before; what's new is a dual type scale and a full component catalog for dense application UI. PROSE (language you read — agent responses, headings, descriptions, in Inter) stays generous; CHROME (UI you scan — labels, data, metadata, badges, in Rajdhani/Mono) is compact. Every value below is a real, rendered size — judge it directly.

Tokens shown with a yellow note are the fixes this pass applies (Theme.fs text-dim was wrong; code-bg / code-border / rail were undocumented; body was missing from the palette).

Dual type scale
The single highest-leverage change. Prose roles stay readable; chrome roles tighten to ~9–11px. The six roles are implemented once as Clavis.Text.* styles (they don't exist today), so the scale is enforced, not hardcoded per control.

Color
Unchanged palette. Two greys for text, two accents for meaning (primary = state, secondary = identity), three signals. The body tone joins the table; the three implementation-only tokens are documented.
Lines & focus
One 1px line (#5A5A66), used only when separation is unavoidable. Focus recolors the same line to primary + a soft 12px glow, animated 250ms. One divider color everywhere; row separators are the same line at 28% opacity.

Iconography
Geometric vector glyphs - straight strokes, square caps, mitred joins, ~2.4 stroke. One meaning per icon. Status is always a circle; "close" is a lighter, neutral cross so it never reads as "problem".

Motion
One duration governs the app: 250ms, CubicEaseOut, for everything that appears, disappears, hovers or focuses. The breathing pulse (600ms, SineEase) is reserved for live status dots. Lists may stagger their row entrances (each row 250ms, total capped). The dedicated motion explorations live in clavis-send-animation.html (rail charge) and clavis-fall-animations.html (gravity falls + the growing turn rail).
