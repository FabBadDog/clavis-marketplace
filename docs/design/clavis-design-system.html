Accent semantics · what the two accents mean

Two accents only earn their keep if each carries a consistent meaning. The split: blue = state, periwinkle = identity. Learn it once, read it everywhere.

Primary · STATE

"Is this active, good, or clickable right now?"
  • Focused window / panel / input (the line goes blue)
  • Success - the good check; passing states
  • Links & live actions; the breathing live-turn dot
  • Markdown H1 (the live title of the doc)

Secondary · IDENTITY

"What is this thing called?" - the unique handle of something.
  • Keyboard gestures & shortcut hints
  • Commit hashes, symbol names, file paths, @mentions
  • Window-identity labels; named sections - markdown H3
  • The "kind" of a thing in a list (command kind, panel kind)

Animation

One transition duration - 250ms, CubicEaseOut - for everything that appears, disappears, hovers, or takes focus (Instant/Quick are gone). Breathing pulse stays at 600ms SineEase. Plus the text reveal below.
Text reveal · char wave → cinematic line-fall
0.8s
1.2k
Summarise the deployment plan.

≤ 3 lines: char-by-char wave - the per-char step shrinks as the text grows, so 2-3 line passes stay as snappy as one line. > 3 lines: cinematic mode - the first line slides in from above, then every other line falls out from behind it into place, staggered. The stagger is capped by line-count, so even 2000 chars finishes in under 2s. The Markdown (10k) button stress-tests it with mixed headers, paragraphs, lists and code - still under 2s.

Motion tokens

  • Standard 250ms CubicEaseOut - all transitions: hover, fade, focus, entrance, dock, tabs
  • Breathing 600ms SineEase - active turn / phase pulse (circles only)
  • Text reveal - char wave (≤3 lines) or line-fall (>3 lines), length-scaled

Row entrance

  • Every row (turn, tool, hook…) enters with fade + slide-up (250ms).
  • Text inside then runs the reveal - no more "poof".
  • Disappearance animates too (fade + slight scale, 250ms).

Where it lives

Tokens

  • Styles.xaml (wpf-host/Theme) - palette brushes, line, tint, surface, font keys, 6 text-role styles.
  • Theme.fs (clavis-rendering) - same tokens for code-built UI.

Widgets

  • clavis-controls - inputs, buttons, labels, badges, dots point at the roles.
  • clavis-rendering - MarkdownPresenter (H1/H3/link accents), SegmentedSelector.
  • New shared Icon factory - good / problem / warning / chevron / dot as vector paths.
